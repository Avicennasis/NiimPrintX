## CRITICAL Findings

### C1. V2 Protocol Routing Incomplete in GUI
**Files:** `PrinterOperation.py:47`, `command.py:128-135`
**Issue:** GUI routes only `"b1"` to `print_imageV2`. CLI correctly routes `("b1", "b18", "b21")`. B18/B21 users get V1 protocol from GUI — prints will fail or produce garbage.
**Fix:** Change to `if self.config.device in ("b1", "b18", "b21"):` — or better, define `V2_MODELS = frozenset({"b1", "b18", "b21"})` in `printer.py` and import everywhere.

### C2. Image Export Anchor Mismatch — Every Print Has Wrong Offset
**Files:** `ImageOperation.py:28-32`, `PrintOption.py:152-158`
**Issue:** `load_image` places images with `anchor="center"` but `export_to_png` reads `canvas.coords()` as top-left anchor. Every exported PNG/printed label has images shifted right/down by half their dimensions.
**Fix:** Switch `load_image` to `anchor="nw"` with adjusted coordinates, matching how text items work.

### C3. toolbar_print_button Permanently Disabled After Closing Preview
**Files:** `PrintOption.py:92, 279`
**Issue:** `display_print` disables the toolbar Print button. Only `_print_handler` re-enables it. Closing the preview popup without printing never re-enables it — Print button is dead forever.
**Fix:** Wire both `close_button.command` and `popup.protocol("WM_DELETE_WINDOW")` to a handler that re-enables the button.

### C4. Windows GUI Spec Missing All Critical Hiddenimports
**Files:** `spec_files/ui_app/NiimPrintX-windows.spec:42`
**Issue:** Only has `['tkinter']`. Missing bleak, wand, PIL, sv_ttk, platformdirs. Windows GUI builds will crash with `ModuleNotFoundError` at runtime.
**Fix:** Match Linux spec: `collect_submodules('PIL') + collect_submodules('bleak') + collect_submodules('wand') + ['platformdirs', 'sv_ttk']`.

### C5. `cairo` (pycairo) Not in Hiddenimports on Any Platform
**Files:** All 3 UI spec files, `PrintOption.py:10`
**Issue:** `import cairo` is unconditional at top level, but no spec includes it. PyInstaller won't bundle the native bindings correctly.
**Fix:** Add `['cairo']` to hiddenimports on all platforms AND guard the import with try/except matching the wand pattern.

### C6. Poetry Version Mismatch — CI Builds Likely Broken
**Files:** All CI workflow files
**Issue:** `poetry.lock` is v2.1 format (generated by Poetry 2.3.3) but CI installs `poetry==1.8.3`. Poetry 1.x cannot read v2.1 lock files.
**Fix:** Update all workflows to `pipx install poetry==2.3.3`.

### C7. `poetry install` Skips GUI Extras in Build Workflows
**Files:** `_build-linux.yaml:53`, `_build-macos.yaml:45`, `_build-windows.yaml:38`
**Issue:** Plain `poetry install` doesn't install optional `gui` extras. `collect_submodules('wand')` runs against empty environment.
**Fix:** Change to `poetry install --all-extras`.

### C8. ImageMagick Downloaded Without Integrity Check
**Files:** `_build-windows.yaml:46`
**Issue:** Binary ZIP downloaded from imagemagick.org with no SHA-256 verification. Supply chain injection vector.
**Fix:** Add hash verification after download.

### C9. `config.device` Is `None` at Startup — Multiple Crash Paths
**Files:** `AppConfig.py:21`, `PrintOption.py:217,316`
**Issue:** `self.device = None` initially. Multiple callers do `label_sizes[config.device]` without None guard.
**Fix:** Initialize `self.device = next(iter(self.label_sizes))`.

### C10. Heartbeat Coroutine Fires Before async_loop Is Running
**Files:** `main.py:32`, `PrintOption.py:29`
**Issue:** Background asyncio thread may not have called `run_forever()` before `PrintOption.__init__` submits the heartbeat coroutine. Silently never executes.
**Fix:** Add `threading.Event` to signal loop readiness; wait before creating widgets.

### C11. `canvas_click_handler` KeyError on Deleted Items
**Files:** `CanvasOperation.py:10-11`
**Issue:** Accesses `text_items[current_selected]` with no membership guard. Rapid deletion or timing issues cause unhandled `KeyError`.
**Fix:** Add `if current_selected in text_items:` guard.

### C12. `merge_label_sizes` Mutates Input Dict In-Place
**Files:** `UserConfig.py:67,87-92`
**Issue:** Function modifies `builtin_sizes` directly and returns it. Multiple `AppConfig` instantiations (tests) accumulate stale state.
**Fix:** Deep-copy at function entry.

### C13. CI Currently Broken — 29 Ruff Errors + 28 Formatting Issues
**Files:** All Python files
**Issue:** Every push to main/dev produces red CI. The lint and format checks both fail.
**Fix:** Run `ruff check --fix . && ruff format .` and fix remaining manual issues.

### C14. Windows Build Has No Tag Validation
**Files:** `_build-windows.yaml:17-23`
**Issue:** No validation that VERSION is non-empty. Artifact names become `NiimPrintX-App--Windows-X64.zip`.
**Fix:** Add validation step matching Linux workflow.

### C15. `tag.yaml` Missing Top-Level Permissions
**Files:** `tag.yaml`
**Issue:** No top-level `permissions:` block. Ambient token permissions leak.
**Fix:** Add `permissions: contents: read`.

### C16. `_create_icon_widgets` Called After Frame Destroyed
**Files:** `TabbedIconGrid.py:116-134`
**Issue:** Background thread finishes loading, `after(0)` fires on destroyed frame widget. Unhandled `TclError`.
**Fix:** Check `frame.winfo_exists()` at start of `_create_icon_widgets`.

### C17. `delete_bounding_box` Doesn't Reset bbox/handle to None
**Files:** `TextOperation.py:200-203`
**Issue:** Stale integer IDs left in dict after canvas items deleted. Subsequent operations on stale IDs corrupt canvas state.
**Fix:** Set `bbox = None` and `handle = None` after deletion.

### C18. Decompression Bomb Check Fires After `.load()`
**Files:** `FileMenu.py:138-142, 162-173`
**Issue:** Manual pixel check runs after `Image.open().load()` — memory already allocated. The real guard (MAX_IMAGE_PIXELS) is set, but manual post-load checks are too late for formats PIL doesn't bomb-check.
**Fix:** Move size checks between `Image.open()` and `.load()`.

## Architecture Recommendations

1. **Extract `V2_MODELS`, `B_SERIES_MAX_WIDTH`** constants to `nimmy/printer.py` — single source of truth
2. **Split `AppConfig`** into immutable config + mutable `CanvasState` with methods instead of raw dicts
3. **Move `helper.py`** from `nimmy/` to `cli/` — it's a Rich presentation layer, not core
4. **Move `UserConfig.py`** from `ui/` to `nimmy/` — it has no UI dependency
5. **Extract `mm_to_pixels`** to shared utility — currently duplicated in CanvasSelector and PrintOption
6. **Use callbacks in FileMenu** instead of reaching into `root.text_tab.text_op.select_text`
7. **Define TypedDicts** for `font_props`, `text_items`, `image_items`, heartbeat/RFID responses
